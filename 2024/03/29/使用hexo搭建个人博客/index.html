使用hexo搭建个人博客

1. 第一步—安装Node.js

因为我们使用的博客框架hexo是基于node.js生成的，是需要node.js支持的，所以我们要先配置node.js环境

【Node.js是一个基于Google的V8JavaScript引擎的开源服务器环境，用于开发服务器端和网络应用程序。Node.js允许使用JavaScript进行服务器端编程，使得开发者能够用同一种语言构建前后端应用，减少了学习成本和代码重复性。Node.js的特点包括事件驱动和非阻塞I/O模型，能够处理大量并发连接，适合构建实时应用和高吞吐量的网络服务。Node.js提供了丰富的模块库，简化了Web应用程序的开发，且其高性能、高并发处理能力使其成为现代网络应用开发的重要工具之一。】

安装好后它会自动安装两个软件到电脑里，一个是node.js，一个是npm

【npm是Node Package Manager的缩写，即Node包管理器，是Node.js的一个包管理和分发工具。npm是Node.js平台的默认包管理工具，也是世界上最大的软件注册表，包含超过600000个包的结构，允许用户轻松跟踪依赖项和版本。npm是一个开放源代码的命令行工具，用于安装、更新和管理Node.js模块。它允许用户从一个集中的仓库中下载和安装公共的Node.js模块。npm随Node.js一起安装，解决了Node.js代码部署中的许多问题，如包的管理（包括安装、卸载、更新、查看、搜索、发布等），并详细记录了每个包的信息，包括作者、版本、依赖、授权信息等，从而将开发者从繁琐的包管理工作（如版本、依赖等）中解放出来，更加专注于功能的开发。】

我们进入命令行查看安装的node和npm的版本

打开终端命令：cmd + space 唤起搜索框，然后输入 terminal

进入root模式：sudo su

查看node版本：node -v

查看npm版本：npm -v

2.第二步-安装hexo博客框架

使用npm安装hexo：npm install -g hexo-cli

【hexo-cli是一个hexo的脚手架（即hexo框架）】

安装完成后查看hexo版本：hexo -v

3.第三步-使用hexo搭建博客

首先建一个项目相关文件夹

查看当前所在文件位置：pwd

在当前文件目录下新建文件夹：mkdir blog

查看当前目录下所有文件夹确认是否新建成功：ls

进入新建的blog目录下：cd blog

新建一个hexo项目：sudo hexo init

启动这个项目：hexo s（完整命令是：hexo server）

在本地访问这个项目：在浏览器网址栏输入localhost:4000

断开服务（停止项目）：ctrl + c

新建一篇文章：hexo n “blog name”（完整命令是：hexo new “blog name”）

4.第四步-将博客部署到github

登陆github，新建一个仓库（repository）

完成后打开终端，进入blog文件目录下，下载一个git插件：npm install –save hexo-deployer-git

在blog文件中有一个_config.yml文件，存放了hexo的相关设置，用vim模式打开它：vim _config.yml

按下 i 进入insert模式，找到文件底部deploy模块，设置如下：

deploy:
type: git
repo: 复制粘贴新建的git仓库的URL
branch: master

然后按esc键，再输入 :wq 保存并退出

部署hexo到github上：hexo d

会提示输入github的用户名和密码，用户名正常输入，密码输入你在github上创建的个人token（因为github为了安全现在不支持直接用用户名密码进行api远程登录）

部署成功后在仓库中可以看到最新的博客代码

进入github—>settings—>pages，source选择deploy from a branch，branch选择master，点击save，这样存放在仓库中的代码才会在github上构建并部署

最后在浏览器中输入仓库名就可以直接访问到博客页面了
